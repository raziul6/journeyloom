Byteflows Travel & Hotel Booking
Documentation

Everything you need to install, configure, customize and extend the complete travel & hotel booking platform for WordPress — free core and Pro add-on.

📦 Plugin v1.0.0 · Requires WP 6.0+ · PHP 7.4+

📥 Installation

From the WordPress dashboard

  1. Open the plugin installer Go to Plugins → Add New and search for Byteflows Travel & Hotel Booking.
  2. Install & activate Click Install Now, then Activate.
  3. Run the Setup Wizard On activation you're taken to the guided setup — or find it later under Byteflows Travel → Setup Wizard.

Manual upload

  1. Download byteflows-travel-hotel-booking.zip.
  2. Go to Plugins → Add New → Upload Plugin, choose the zip and click Install Now.
  3. Activate the plugin.
ℹ️

On activation the plugin creates its database tables (bookings, wishlist), seeds default taxonomy terms, and auto-creates the system pages. Deactivating never deletes your data.

📑 Duplicating Content

Hover any trip or hotel in its list table and click Duplicate. The copy includes all meta (pricing, itinerary, rooms, galleries…) and taxonomy assignments, and is created as a draft so you can edit before publishing. Great for building seasonal variants of the same package.

📆 Bookings & Statuses

All bookings are stored in the plugin's own table and managed under Byteflows Travel → Bookings.

The booking flow

  1. Customer picks dates & travelers The booking widget (on single trip/hotel pages or via [wptm_booking_form]) checks live availability via AJAX. Trips with pricing tiers show +/− quantity steppers per tier (Adult, Child, Group…); the total updates live. Hotel bookings are per room and stay dates.
  2. Traveler details Name, email, phone and any notes are collected in the "Your Details" step.
  3. Payment Bank transfer instructions (free) or online checkout (Pro). Totals are re-calculated server-side from the stored prices — client-side price tampering is impossible.
  4. Confirmation Customer lands on the confirmation page and receives an email; you get an admin notification and a red counter bubble on the WP menu.

Cart checkout (multi-item)

Instead of booking one item at a time, customers can add to cart multiple trips/rooms, review them on the Cart page ([wptm_cart]), apply a coupon code Pro, and pay for everything in one checkout. Cart totals are also validated server-side.

Statuses

StatusMeaning
PendingNew booking awaiting review/payment. Counted in the admin menu bubble.
ConfirmedApproved — customer is notified by email on the status change.
CompletedThe trip/stay has taken place.
CancelledCancelled by you or the customer; excluded from revenue.

Manager & reports

Click any row to open the slide-in drawer with full booking details, traveler list and status controls. Byteflows Travel → Reports tracks bookings and revenue over time; the Dashboard shows totals at a glance.

💳 Payments

Manual / Bank Transfer Free

Enable it under Settings → Manual Payment and write your payment instructions (bank account, deadline, reference format…). They're shown at checkout and in the confirmation email; you confirm the booking manually once paid.

Online gateways Pro

The Pro add-on adds Stripe (cards with SCA/3-D Secure), PayPal, and Razorpay (cards, UPI, netbanking, wallets) — all verified server-side. See Pro → Online Payments.

ℹ️

Developers can register additional gateways with the wptm_payment_gateways filter — see Hooks.

✉️ Email Notifications

Configured under Settings → Notifications:

  • Admin notification — sent to your booking email address for every new booking (toggle on/off).
  • Customer confirmation — sent on booking, with a customizable subject line (toggle on/off).
  • Status updates — customers are emailed when you confirm or cancel their booking.
  • Sender controls — from-name, from-address and a footer text appended to every email.
💡

Emails are sent through wp_mail(), so any SMTP plugin (WP Mail SMTP, FluentSMTP…) improves deliverability automatically.

❤️ Wishlist, Compare & Reviews

Wishlist

Visitors (guests included) heart trips/hotels; favorites persist and display on the Wishlist page. Toggle sitewide under Settings → Display.

Compare

Add 2+ items to comparison and a floating compare bar appears at the bottom of the screen showing the selected items; open the compare view for a side-by-side table of price, duration, rating, facilities and more — built automatically from item data.

Reviews & ratings

Star-rated reviews on trips and hotels using the WordPress comment system, so all moderation tools apply (Comments screen, spam plugins…). Toggle under Settings → Display.

Recently viewed & related

A recently-viewed strip and a "You may also like" related-items section (toggle + count under Settings → Display) keep visitors browsing.

💬 Enquiry Form

Let visitors ask about a trip before booking. Under Settings → Enquiry Form you can:

  • Control where the form appears.

🎨 Appearance & Branding

Two levels of control:

  • Settings → Appearance — pick your primary, ribbon and icon colors with color pickers. Leave blank for defaults.

The plugin ships a self-hosted Inter font (no CDN requests). Disable it or point to your own with the wptm_enqueue_fonts / wptm_fonts_url filters.

💳 Online Payments Pro

With Pro active, gateway panels appear in Byteflows Travel → Settings. Enable a gateway, paste its keys, and it shows up as a payment option at checkout alongside bank transfer.

Gateway settings

GatewaySettingsHow it works
StripeEnable toggle · Publishable key · Secret key · Webhook secretStripe.js loads only on checkout. Payment intents are created and verified server-side with SCA / 3-D Secure; card numbers never touch your server. Use pk_test_/sk_test_ keys to test.
PayPalEnable toggle · Mode (Live / Sandbox) · Client ID · SecretPayPal buttons render via the JS SDK; orders are created and captured server-side through PayPal's REST API. Switch Mode to Sandbox for testing.
RazorpayEnable toggle · Key ID · Key secret · Webhook secretCards, UPI, netbanking and wallets. An order is created via the Razorpay API and the returned payment signature is verified server-side before the booking is confirmed.

Setup checklist

  1. Get your API keys From the gateway's dashboard (Stripe Dashboard → Developers → API keys; PayPal Developer → Apps; Razorpay Dashboard → API Keys).
  2. Enable & paste keys In the gateway's panel under Byteflows Travel → Settings. Add the webhook secret where offered (Stripe, Razorpay) so asynchronous payment events are verified.
  3. Test a booking Use the gateway's test/sandbox mode and its test cards before going live.
  4. Go live Swap in live keys. The payment status flows into the booking automatically — wptm_payment_completed fires for developers.
⚠️

Online gateways require HTTPS on your checkout page. Currency comes from Settings → Currency — make sure your gateway account supports it.

🏷️ Coupons & Discounts Pro

Pro adds a Coupons screen under the Byteflows Travel menu — a list of your codes with columns for Code, Type, Amount, Usage, Expiry and Status.

Creating a coupon

FieldDescription
CodeWhat the customer types at checkout, e.g. SUMMER20. Case-insensitive.
TypePercentage (e.g. 20% off the total) or Fixed amount (e.g. $50 off).
AmountThe percentage or fixed value.
Usage limitMaximum number of redemptions; usage is counted automatically.
ExpiryDate after which the code stops working.
StatusActive / inactive — pause a code without deleting it.

Customer flow

Customers enter the code in the coupon field on the Cart/Checkout page. It's validated instantly via AJAX, the discount line appears in the totals, and the final amount is re-validated server-side when the booking is created — an expired or exhausted code can't slip through.

📍 Pickup Points Pro

Offer meeting/pickup locations per trip — free or with a surcharge.

  1. Add points in the trip editor A Pickup Points panel appears on every trip: repeatable rows with a label (e.g. "Airport Terminal 1") and an optional price (leave 0 for a free pickup).
  2. Traveler chooses at checkout Each traveler selects their pickup point during booking.
  3. Surcharge added automatically Priced pickups are added to the total — and re-validated server-side like everything else.

The chosen pickup points appear in the booking drawer and on the invoice, so your ground team knows exactly who boards where.

🤖 AI Assistant Pro

Bring-your-own-key AI. Nothing is sent to any provider until you enable AI and enter a key.

Setup

  1. Open the AI panel Byteflows Travel → Settings → AI Features (appears with Pro active).
  2. Turn on "Enable AI" The master switch — nothing is sent anywhere while it's off.
  3. Choose a provider & model OpenAI (default model gpt-4o-mini), Anthropic (default claude-opus-4-8), or Custom — any OpenAI-compatible endpoint by Base URL (Groq, Gemini's compatible endpoint, OpenRouter, self-hosted Ollama…). The model field accepts any model ID your provider offers.
  4. Enter your API key You pay the provider directly; there is no AI subscription from Byteflows.
  5. Toggle the front-end features you want Chat and recommendations each have their own switch.

🎨 CSS Variables

All front-end styling hangs off custom properties on :root. Override them in your theme stylesheet (or the Customizer's Additional CSS) — zero plugin edits:

:root {
    --wptm-primary: #0e7490;   /* brand color */
    --wptm-font: 'Poppins', sans-serif;
    --wptm-radius: 8px;         /* card corner radius */
}

Ribbon, icon and primary colors can also be set visually under Settings → Appearance.

🛠️ Troubleshooting

Single trip/hotel pages show 404

Rewrite rules need a refresh: go to Settings → Permalinks and click Save Changes (no edits needed).

Emails not arriving

Your host's PHP mail is likely unreliable — install an SMTP plugin (WP Mail SMTP, FluentSMTP). The plugin sends via wp_mail(), so it picks up SMTP automatically. Check the from-address domain matches your site for SPF/DKIM.

A system page shows nothing

Make sure the page still contains its shortcode and is mapped under Settings → Pages.

Front end looks unstyled

A caching/optimization plugin may be deferring CSS — exclude handles starting with wptm- from CSS optimization, then clear caches.

Pro panels don't appear

Both plugins must be active: the free core and the Pro add-on. Reactivate Pro if you've just updated the free plugin.

AI features return an error

Check, in order: Enable AI is on and an API key is saved under Settings → AI Features; the key is valid and has credit with your provider; the model ID exists for that provider; for a Custom provider, the Base URL points to an OpenAI-compatible /v1 endpoint. A "Too many requests" message is the built-in rate limiter protecting your API credits — wait a moment and retry.

Online payment fails or stays pending

Confirm the checkout page is HTTPS, the right mode is selected (test vs live keys must match the mode), and the webhook secret is set where offered (Stripe, Razorpay). Gateway dashboards log every attempt with the exact decline reason.
